# Design Pass

Orient all work toward design artifacts: mockups, prototypes, design tokens, and component specs. Production code is out of scope for this pass.

## Builder Focus

- Produce wireframes, mockups, or high-fidelity prototypes depending on the unit's fidelity requirements
- Define or extend design tokens (colors, spacing, typography, elevation) in the project's token system
- Specify component structure, states, and interaction behavior
- Document responsive breakpoints and adaptive layout rules
- Capture micro-interactions and transition specifications

## Reviewer Focus

- Verify designs address every acceptance criterion in the unit
- Check consistency with existing design system tokens and patterns
- Confirm interaction flows cover error states, empty states, and loading states
- Ensure accessibility considerations are documented (contrast, focus order, ARIA semantics)

## Completion Signal

The design pass is complete when design artifacts are reviewed and detailed enough to inform subsequent product or dev passes without ambiguity. No production code is produced.
