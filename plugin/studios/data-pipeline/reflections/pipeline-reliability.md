name: pipeline-reliability
studio: data-pipeline
---

**Analyze:** Pipeline run success rates, failure modes, recovery times, SLA adherence.

**Look for:**
- Most common failure modes (timeout, schema drift, resource exhaustion, source unavailable)
- Mean time to recovery after failures
- Whether retry policies were effective or masked systemic issues
- SLA breaches and their root causes

**Produce:**
- Reliability metrics (success rate, MTTR, SLA adherence)
- Failure mode taxonomy
- Recommendations for retry policy, alerting, and infrastructure changes
