**Purpose:** Aggregate and review user feedback signals to inform prioritization. Anecdotes aggregate into patterns.

**Procedure:**
- [ ] Review support tickets tagged as feature requests
- [ ] Check NPS verbatim responses for product themes
- [ ] Review sales objections related to missing features
- [ ] Aggregate signals by theme and frequency
- [ ] Update opportunity scores if patterns shift

**Signals:**
- Biweekly cadence
- Spike in a particular feature request
- Sales reports recurring objection pattern
