**Purpose:** Identify documentation that has fallen behind the codebase or product.

**Procedure:**
- [ ] Compare documentation against current feature state
- [ ] Check code examples compile/run against current version
- [ ] Identify pages with no updates in 6+ months
- [ ] Flag pages for features that have been significantly changed
- [ ] Prioritize updates by page traffic and user impact

**Signals:**
- Quarterly review
- After major product releases
- After API breaking changes
